Project Name: Alunex Façade PM System

Platform: Static HTML/CSS/JavaScript Wireframe

================================================================================
1. PROJECT OVERVIEW
================================================================================

This project is a complete redesign and enhancement of an existing project
management system wireframe, transforming it from a generic PM tool into a
world-class, industry-specific façade and construction project management
SaaS platform.

INITIAL STATE:
- Basic wireframe with 10 simple navigation items
- Generic project management features
- Limited scope (time tracking, tasks, files, chat, etc.)
- No industry-specific workflows
- Simple flat navigation structure

FINAL STATE:
- 42 comprehensive HTML pages
- 30+ specialized modules across 6 major sections
- Façade and construction industry-specific workflows
- Multi-country/timezone support (UAE, India, China, Oman)
- Role-Based Access Control (RBAC) system
- Advanced Earned Value Management (EVM) analytics
- Collapsible sidebar with organized sections
- Portfolio management dashboards

REQUIREMENT #2: Multi-Country & Timezone Support
------------------------------------------------
Client needed support for global façade projects spanning multiple countries:

SUPPORTED COUNTRIES:
- UAE (United Arab Emirates) - Primary country for financial reporting
- India - Engineering & design teams
- China - Procurement & fabrication
- Oman - Additional fabrication & installation

TIMEZONES:
- GST (Gulf Standard Time) - UAE

CURRENCIES:
- AED (Arab Emirates Dirham) - Primary

A single façade project might have:

- Fabrication in China/Oman
- Installation in UAE
- Financial reporting in AED from UAE


REQUIREMENT #3: Role-Based Access Control (RBAC)
-------------------------------------------------
Client requested a 4-tier permission system with budget visibility controls:

ROLE 1: CEO / Director
- Full financial visibility across ALL projects
- Access to all departments and modules
- Can view commercial details, budgets, and profit margins
- Portfolio-level analytics and reporting

ROLE 2: Project Manager
- Access to ENTIRE project they manage
- Can see all departments within their project
- Budget visibility limited to their project only
- Cannot see other projects' financial data

ROLE 3: Department Head
- Access to their department only (e.g., Engineering, Procurement)
- Budget visibility limited to their department allocation
- Performance metrics for their team
- Cannot see other departments' budgets

ROLE 4: Site Engineers / Team Members
- NO budget or financial access
- Task management and time tracking only
- Document access for their work area
- Communication tools

-----------------------------------------

EVM METRICS REQUIRED:

- SPI (Schedule Performance Index)
- CV (Cost Variance)
- SV (Schedule Variance)
- EAC (Estimate at Completion)
- ETC (Estimate to Complete)
- VAC (Variance at Completion)
- TCPI (To-Complete Performance Index)

DASHBOARD REQUIREMENTS:
- Portfolio financial overview
- Budget performance by project
- Schedule performance tracking

- Multi-project analytics

7. FREQUENTLY ASKED QUESTIONS
================================================================================

Q1: Can we add back some old features like "Video Calls"?
A: Yes! Video Calls functionality is integrated into the Communication module,
   and Jitsi Meet integration is available in Settings. We can add a direct
   link if needed.

Q2: Where is "Time Tracking" now?
A: It's been upgraded to "Man-Hour Tracking" in the MONITORING & CONTROL
   section, which is the construction industry standard term and includes
   integration with budget tracking and cost control.

Q3: What happened to "Task Management"?
A: Task management is now integrated into each project phase module
   (Engineering, Procurement, Fabrication, Installation, etc.) and the
   "Close-Out & Snag" module includes snag list management.

Q4: Is "Department Analytics" gone?
A: No! The department-analytics.html page still exists and was created in
   Phase 1. It's a separate page accessible from the system.

Q5: Can I still do basic PM tasks like the old system?
A: Yes, but now you have industry-specific versions:
   - Time logging → Man-Hour Tracking
   - Tasks → Phase-specific task modules
   - Files → Documents & Drawings
   - Chat → Communication
   All with enhanced capabilities!

Q6: Why so many pages now (42 vs 10)?
A: The client requested comprehensive coverage of the entire façade project
   lifecycle, from tender to close-out, with separate modules for each
   phase and function. This provides better organization and scalability.

Q7: Will this be harder to use with so many modules?
A: No! The collapsible sidebar lets you hide sections you don't use, and
   the hierarchical organization makes it easier to find what you need.
   The RBAC system also shows users only what's relevant to their role.
